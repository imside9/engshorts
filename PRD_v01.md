## ✅ 코덱스 전달용: WordSwipe MVP 스펙 (UI/UX + 문제타입 + 구현 태스크)

### 0) MVP 목표

- **멀리서 보면 쇼츠/릴스 보는 것처럼 보이는 UI**
  
- **3~5연속 정답마다 “보상 체감”이 확 와야 함**
  
- **문제 타입이 계속 바뀌어 지루함이 없어야 함**
  
- 데이터는 폴더 내 CSV(1800단어) 사용
  

---

# 1) UI/UX 요구사항 (숏폼 느낌 강제)

## 1-1. 화면 구조 (릴스/쇼츠 클론 느낌)

- **세로 풀스크린** 카드 1장 = 콘텐츠 1개
  
- 위로 스와이프: 다음 카드 / 아래로 스와이프: 이전 카드(옵션)
  
- 오른쪽에 “쇼츠 느낌” 세로 아이콘 더미(기능 최소):
  
  - 🔥 streak
    
  - ⚡ speed
    
  - 🎁 reward count
    
- 상단에는 **얇은 타이머 바**(progress bar) + 작은 진행 정보(예: `12 / ∞`)
  
- 하단에는 짧은 캡션 텍스트(학습앱 느낌 최소화)
  

## 1-2. 카드 전환 애니메이션 (중요)

- 다음 카드 넘어갈 때 **물리감(bounce) 있는 슬라이드**
  
- 정답: 살짝 확대(예: 1.05) + 글로우
  
- 오답: 좌우 흔들림 + 짧은 플래시
  
- 시간 초과: 자연스럽게 fade/slide 후 다음으로 자동 이동
  

## 1-3. 배경(반복 금지) 시스템 (매 카드마다 변화)

- 카드마다 배경이 바뀌어야 함 (정적 단색 금지)
  
- 배경 소스: 그라데이션/패턴/추상 이미지 등 **최소 60개 이상** 준비(같은 걸 돌려써도 됨)
  
- **최근 N개(예: 10개)와 중복 금지** 로직 필수:
  
  - `pickRandom(excludingLast10Backgrounds)`
- 배경 위에 **은은한 모션(파티클/노이즈/그라데이션 애니메이션)** 있으면 더 숏폼 느낌
  

---

# 2) 타이머/리듬 (퀴즈 카드 핵심 규칙)

## 2-1. 시간 제한

- 퀴즈 카드 기본 제한: **3초**
  
- 0초 되면:
  
  - 자동 오답 처리
    
  - 즉시 다음 카드로 전환(0.8초 이내 리듬 유지)
    
- 타이머는 상단 얇은 바가 줄어드는 형태(숏폼 UI 감성)
  

## 2-2. “머물 수 없게” 만드는 리듬

- 사용자가 고민하며 정지하면 실패 → **강제 진행**이 UX 핵심

---

# 3) 보상 시스템 (3~5연속 정답이면 체감 확)

## 3-1. 보상 트리거

- streak가 **랜덤 3~5**에 도달하면 보상 카드 발생
  
  - `if (streak >= randomInt(3,5)) triggerReward()`
- 또는 안전장치로: 10문제 진행 시 최소 1회 보상 보장
  

## 3-2. 보상 카드 연출(무조건 강하게)

- 보상 카드 진입 시:
  
  - 배경 완전 교체 + 밝기 순간 상승
    
  - 큰 타이포(“Nice🔥”, “3연속 성공!”)
    
  - (가능하면) 짧은 사운드/진동
    
- 보상 콘텐츠(MVP):
  
  1. 5~10초 세로형 내부 영상(또는 애니메이션 배경)
    
  2. 동기부여 문구 카드
    
  3. “스피드 보너스” 카드(맞히면 streak 2배)
    

※ 유튜브 iframe은 **사용하지 않음**

---

# 4) 문제 타입 다양화 (지루함 방지 핵심)

## 4-1. MVP 최소 6종 문제 타입 포함

1. **단어→뜻 4지선다** (기본)
  
2. **뜻→단어 4지선다** (역방향)
  
3. **OX 초스피드** (2초)
  
4. **빈칸 문장 완성** (4지선다)
  
5. **초간단 보너스** (너무 쉬운 문제, 맞히면 streak 2배)
  
6. **스피드 선택** (여러 단어 중 빠르게 선택, 2초)
  

## 4-2. 타입 반복 금지 규칙

- 같은 타입 **3번 연속 금지**
  
- 같은 단어가 **같은 타입으로 재등장 금지**(가능하면)
  
- 타입 선택은 완전 랜덤이 아니라 “리듬 패턴 + 랜덤”:
  
  - 예시 흐름: `기본 → 역방향 → OX → 기본 → 쉬운 → (보상 체크) → 빈칸 → 기본 …`

## 4-3. 타입 변화가 “화면 장르 변화”처럼 보이게

- 타입 바뀔 때마다:
  
  - 폰트 크기/정렬/카드 레이아웃 조금씩 다르게
    
  - 타이머 길이 다르게(2초/3초)
    
  - 배경 톤도 다르게
    

---

# 5) 데이터/콘텐츠 처리 (CSV 활용)

## 5-1. 입력

- 폴더 내 CSV 여러 개 로딩 가능
  
- 로딩 후 내부에서 JSON 형태로 변환, 셔플하여 무한 피드 제공
  

## 5-2. 단어 상태 관리(반복 방지에 필요)

- 최근 단어 큐(예: last 30개) 저장 → 재등장 제한
  
- 최근 배경 큐(예: last 10개) 저장 → 중복 금지
  
- lastShownType 저장 → 타입 반복 제한
  

---

# 6) 개발 태스크(우선순위)

P0 (MVP 필수)

- CSV 파서 + 단어 풀 구성 + 셔플
  
- 카드 피드/스와이프 UI + 전환 애니메이션
  
- 타이머 바 + 시간초과 자동 다음
  
- 문제 타입 6종 생성기 + 타입 반복 방지
  
- 배경 시스템(60+ 에셋) + 최근 중복 방지
  
- 보상 트리거(3~5 랜덤) + 강한 연출
  

P1 (가능하면)

- 진동/짧은 효과음 옵션
  
- 문제 결과에 따라 미세한 난이도 조절(쉬운 카드 빈도 조정)
